# 缺陷清单

## 1. 缺陷记录说明

字段说明如下：

- 缺陷ID：唯一编号
- 模块：问题所属模块
- 严重级别：`S1` 阻断 / `S2` 严重 / `S3` 一般 / `S4` 轻微
- 优先级：`P0` 最高 / `P1` 高 / `P2` 中 / `P3` 低
- 状态：已修复 / 已验证
- 复现步骤：可稳定重复的问题路径
- 原因分析：定位到的真实原因
- 修复方案：采取的处理方式
- 回归结果：修复后的验证结论

## 2. 代表性缺陷

### BUG-001 地面命中时重复扣分

- 模块：子弹结算 / 计分逻辑
- 严重级别：S2
- 优先级：P0
- 状态：已修复，已验证
- 复现步骤：
  1. 进入游戏并点击开始。
  2. 朝地面发射子弹。
  3. 观察总分和浮字显示。
- 实际结果：单次命中地面会触发两次扣分，出现 `-20` 或连续两次 `-10`。
- 预期结果：单次命中地面只扣一次 10 分。
- 原因分析：同一发子弹在 `OnHit`、`OnProjectileStop`、`Destroyed` 等链路中发生重复结算。
- 修复方案：收敛 miss 扣分入口，并引入 `HasResolved` 防止单发子弹重复结算。
- 回归结果：连续多次命中地面均只扣一次 10 分。

### BUG-002 命中目标后仍被误判为 miss

- 模块：子弹结算 / 目标命中
- 严重级别：S2
- 优先级：P0
- 状态：已修复，已验证
- 复现步骤：
  1. 进入游戏。
  2. 命中任意目标。
  3. 观察总分和浮字变化。
- 实际结果：命中目标后先加分，随后又出现 `-10`，导致净分异常。
- 预期结果：命中目标只应加分，不应再触发 miss 扣分。
- 原因分析：命中目标和 miss 分支的时序冲突，子弹销毁后仍继续进入 miss 逻辑。
- 修复方案：命中目标时优先设置 `HasResolved = true`，其他结算分支进入前先检查状态位。
- 回归结果：命中目标后仅执行加分逻辑，未再发生误扣分。

### BUG-003 子弹命中后短暂停留在地面或目标表面

- 模块：子弹生命周期 / 碰撞
- 严重级别：S3
- 优先级：P1
- 状态：已修复，已验证
- 复现步骤：
  1. 开始游戏。
  2. 分别朝地面、障碍物、目标开火。
  3. 观察子弹命中后的销毁表现。
- 实际结果：部分场景中子弹短暂停留在表面，出现“黏地”现象。
- 预期结果：子弹命中后应及时销毁，不应残留。
- 原因分析：碰撞事件与停止事件的执行顺序不稳定，销毁链路不统一。
- 修复方案：统一有效结算后的销毁时机，并调整相关碰撞处理逻辑。
- 回归结果：命中地面、障碍物和目标后均可正常销毁。

### BUG-004 目标数量随着击毁次数递减

- 模块：目标补位 / 生成管理
- 严重级别：S2
- 优先级：P0
- 状态：已修复，已验证
- 复现步骤：
  1. 连续击毁场上目标。
  2. 观察目标补位情况。
- 实际结果：目标不会稳定补位，场上目标数量会逐渐减少。
- 预期结果：每次击毁后都应立即补位，场上始终维持 5 个目标。
- 原因分析：部分目标实例中的 `ManagerRef` 丢失，导致无法回调补位管理器。
- 修复方案：目标生成后立即写入 `ManagerRef`，并在关键节点增加有效性检查。
- 回归结果：连续击毁 30 次后，场上仍稳定维持 5 个目标。

### BUG-005 目标刷到 SpawnArea 外

- 模块：随机生成
- 严重级别：S3
- 优先级：P1
- 状态：已修复，已验证
- 复现步骤：
  1. 配置 SpawnArea。
  2. 连续击毁目标触发多次补位。
  3. 观察新目标刷出位置。
- 实际结果：部分目标刷到区域外或出现在不合理高度。
- 预期结果：目标应全部生成在指定区域内。
- 原因分析：世界坐标中心、`Box Extent` 和固定高度计算混用，导致随机点偏移。
- 修复方案：重新梳理中心点、范围和固定 Z 的关系，修正最终生成坐标。
- 回归结果：连续补位未再出现刷出区域的问题。

### BUG-006 三色目标分值正确但材质显示异常

- 模块：目标表现 / 材质配置
- 严重级别：S3
- 优先级：P1
- 状态：已修复，已验证
- 复现步骤：
  1. 进入游戏。
  2. 观察三色目标外观并分别命中。
- 实际结果：分值按 10 / 20 / 30 结算，但目标材质显示异常。
- 预期结果：目标颜色与分值映射一致。
- 原因分析：材质变量未正确绑定到目标对象。
- 修复方案：补齐默认材质赋值，并在分值分支中显式调用材质设置。
- 回归结果：目标表现和实际分值保持一致。

### BUG-007 准星居中但子弹轨迹明显偏移

- 模块：射击对齐
- 严重级别：S3
- 优先级：P1
- 状态：已修复，已验证
- 复现步骤：
  1. 使用准星瞄准中远距离目标。
  2. 连续开火并观察落点。
- 实际结果：准星在屏幕中心，但子弹落点明显偏下。
- 预期结果：弹道应与准星基本对齐。
- 原因分析：瞄准基于相机中心，发射基于枪口位置，导致视差明显。
- 修复方案：先通过相机 `LineTrace` 获取 `AimPoint`，再让枪口朝 `AimPoint` 发射。
- 回归结果：中远距离弹道与准星基本一致，近距离仅保留可接受的轻微视差。

### BUG-008 重开后角色无法移动和转动视角

- 模块：重开流程 / 输入模式
- 严重级别：S2
- 优先级：P0
- 状态：已修复，已验证
- 复现步骤：
  1. 完成一局进入结算页面。
  2. 点击 Restart。
  3. 新一局开始后尝试移动和开火。
- 实际结果：倒计时开始运行，但角色无法移动、转向或开火。
- 预期结果：重开后应恢复完整游戏输入。
- 原因分析：关闭 UI 后未从 `UI Only` 切回 `Game Only`。
- 修复方案：在重开链路中补齐输入模式与鼠标状态恢复逻辑。
- 回归结果：从结算页和暂停页重开后均可直接恢复游戏控制。

### BUG-009 首次开始与重开自动开始逻辑冲突

- 模块：开始流程 / 状态门控
- 严重级别：S3
- 优先级：P1
- 状态：已修复，已验证
- 复现步骤：
  1. 首次进入关卡并点击开始。
  2. 完成一局后点击 Restart。
  3. 观察下一局起始状态。
- 实际结果：重开后仍要求再次点击“开始游戏”。
- 预期结果：首次进入需要点击开始，重开后应直接自动开始。
- 原因分析：首次启动门控和重开流程复用了同一套初始暂停逻辑，缺少状态区分。
- 修复方案：新增 `BP_MyGameInstance.AutoStartAfterRestart` 标记，在 `BeginPlay` 中分流处理。
- 回归结果：首局需要点击开始，后续重开可自动开局。

### BUG-010 ESC 暂停菜单无法稳定关闭或恢复输入

- 模块：暂停流程
- 严重级别：S3
- 优先级：P1
- 状态：已修复，已验证
- 复现步骤：
  1. 游戏中按 `ESC` 打开暂停菜单。
  2. 再按 `ESC` 或关闭菜单。
  3. 观察菜单关闭和控制恢复情况。
- 实际结果：部分场景中暂停菜单无法通过 `ESC` 正常关闭，或关闭后输入状态不完整。
- 预期结果：`ESC` 应能稳定打开和关闭暂停菜单，恢复后角色控制正常。
- 原因分析：暂停状态下输入事件未完整配置，且打开、关闭菜单时的 UI 和输入切换不对称。
- 修复方案：开启“暂停时可执行”输入，并统一菜单开关时的暂停、UI、输入模式和鼠标状态切换。
- 回归结果：`ESC` 菜单开启、关闭和恢复控制均稳定。
