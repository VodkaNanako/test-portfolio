# 核心展示版测试用例清单

## 1. 测试执行信息

- 测试版本：v1.0
- 引擎版本：UE4.27
- 测试环境：Windows / PIE
- 测试结论：主流程和关键异常流程通过

## 2. 说明

本文档为作品集展示使用的核心测试用例清单，重点展示测试思路、风险识别和关键场景覆盖情况。

为保证阅读效率，这里只保留最能体现测试能力的代表性用例，覆盖以下三类重点内容：

- 主流程验证
- 高风险异常流程验证
- 历史问题的重点回归场景

## 3. 用例设计说明

本轮用例以“可演示、可回归”为目标，优先覆盖以下风险：

- 首次开局与重开流程
- 计分与扣分链路
- 目标生成与补位
- HUD、暂停、结算等关键 UI
- 射击对齐与输入模式切换

优先级说明：

- `P0`：阻断主流程或导致核心规则错误
- `P1`：高风险功能问题，明显影响体验
- `P2`：中风险问题，影响展示效果但不阻断主流程

## 4. 核心展示版测试用例

| 用例ID | 模块 | 优先级 | 测试场景 | 操作步骤 | 预期结果 | 结论 |
|---|---|---|---|---|---|---|
| START-001 | 开始流程 | P0 | 首次进图不点击开始按钮 | 进入关卡后保持不操作 | 游戏保持暂停，角色不可移动，倒计时不开始 | Pass |
| START-002 | 开始流程 | P0 | 首次进图点击开始按钮 | 点击“开始游戏”按钮 | 游戏解除暂停，角色可移动可开火，开始按钮隐藏 | Pass |
| RESTART-001 | 重开流程 | P0 | 结算页重开 | 在结算页面点击 Restart | 关卡重载后直接开始新一局，无需再次点击开始 | Pass |
| RESTART-002 | 重开流程 | P1 | 暂停页重开 | 打开暂停菜单后点击 Restart | 关卡重载后直接进入可操作状态 | Pass |
| SCORE-001 | 计分逻辑 | P0 | 命中目标正常加分 | 命中任意目标一次 | 单次命中只结算一次加分，不重复计算 | Pass |
| SCORE-002 | 计分逻辑 | P0 | 地面命中扣分回归 | 朝地面发射子弹 | 单次只扣 10 分，不出现双扣分 | Pass |
| SCORE-003 | 计分逻辑 | P0 | 命中目标后误扣分回归 | 命中目标后观察总分和浮字 | 命中目标后只加分，不再误触发 miss 扣分 | Pass |
| HUD-001 | HUD 显示 | P1 | 命中反馈显示 | 命中 10 分目标 | 总分增加 10，浮字显示 `+10` | Pass |
| HUD-002 | HUD 显示 | P1 | miss 扣分反馈 | 打空或命中非目标 | 扣 10 分并显示 `-10` 浮字 | Pass |
| SPAWN-001 | 目标生成 | P0 | 开局目标数量 | 开局后观察场上目标 | 初始稳定生成 5 个目标 | Pass |
| SPAWN-002 | 目标生成 | P0 | 目标补位回归 | 击毁 1 个目标后观察补位 | 立即补位，场上目标数量持续维持为 5 | Pass |
| SPAWN-003 | 目标生成 | P1 | SpawnArea 有效性回归 | 连续击毁目标 30 次 | 目标均在有效区域内，不刷到区域外 | Pass |
| TARGET-001 | 目标分值 | P1 | 三色目标映射 | 分别击中三色目标 | 目标颜色与 10 / 20 / 30 分值映射一致 | Pass |
| AIM-001 | 射击对齐 | P1 | 准星与弹道对齐 | 远距离瞄准目标并开火 | 子弹轨迹与准星基本对齐，近距允许轻微视差 | Pass |
| PAUSE-001 | 暂停流程 | P0 | ESC 打开暂停菜单 | 游戏进行中按 `ESC` | 弹出灰屏暂停菜单，游戏暂停 | Pass |
| PAUSE-002 | 暂停流程 | P0 | ESC 关闭暂停菜单 | 暂停菜单打开后再按 `ESC` | 菜单关闭，游戏恢复，角色控制正常 | Pass |
| INPUT-001 | 输入模式 | P0 | 重开后输入恢复回归 | 结算页点击 Restart 后尝试移动、转向、开火 | 重开后输入模式恢复正常，角色可控制 | Pass |
| END-001 | 结算流程 | P0 | 倒计时结束结算 | 等待倒计时归零 | 弹出结算页面，显示最终分数 | Pass |
| END-002 | 结算流程 | P1 | 结算页交互 | 打开结算页面并检查 Restart | 按钮可点击，文本无遮挡，交互正常 | Pass |
