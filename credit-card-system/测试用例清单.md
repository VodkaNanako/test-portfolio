# 核心展示版测试用例清单

## 1. 测试执行信息

- 测试对象：信用卡管理系统
- 测试环境：Windows / Java 17 / Spring Boot / SQLite / Chrome
- 测试重点：真实历史缺陷回归、核心业务规则、账单月隔离、高频操作回归

## 2. 说明

本文档为作品集展示使用的核心测试用例清单，重点展示测试思路、业务风险识别和关键场景覆盖情况。

为保证阅读效率，这里只保留最能体现测试能力的代表性用例，覆盖以下三类重点内容：

- 主流程验证
- 高风险业务规则验证
- 真实历史缺陷回归场景

因此，这份文档更偏向“测试展示材料”，而不是逐条记录执行细节的测试执行台账。

## 3. 用例设计说明

本轮核心展示版用例优先覆盖以下风险：

- 发布包启动与基础访问是否正常
- 页面按钮、弹窗和输入框是否稳定
- 账单月切换与账单月隔离是否正确
- 分类判定和状态联动是否正确
- 刷卡、还款、撤销后的金额联动是否完整
- 操作时间、导出和缓存相关问题是否回归通过

优先级说明：

- `P0`：阻断主流程或导致核心业务结果错误
- `P1`：高风险功能问题，明显影响业务可信度
- `P2`：中风险问题，影响使用体验但不阻断主流程
