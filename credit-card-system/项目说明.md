# 项目说明

## 1. 项目概述

- 项目名称：信用卡管理系统
- 项目类型：本地 Web 管理系统
- 运行方式：浏览器访问 + 本地 SQLite 数据库存储
- 目标场景：替代 Excel 手工管理信用卡账单、刷卡、还款和账单导出

## 2. 核心业务规则

### 2.1 金额计算规则

- 未还金额 = `max(还款额度 + 临时超额, 已刷金额) - 已还金额`
- 未刷金额 = `已还金额 - 已刷金额`

### 2.2 分类判定规则

- 未结算卡：未还金额 = 0 且 未刷金额 != 0
- 已还完卡：未还金额 = 0 且 未刷金额 = 0 且 收费状态 = 已收费
- 强制已还完：可无视金额条件直接进入已还完卡

### 2.3 账单月隔离规则

- 同一卡在不同账单月下，收费状态、强制已还完、额度类字段可以独立维护
- 查询某个月份时，不应被其他月份的配置和日志污染

### 2.4 VIP 规则

- VIP 按“卡主”全局生效，不分账单月
- 应收费 = 还款额度 × 费率
- 普通用户费率为 1.5%，VIP 用户费率为 1%

## 3. 主要模块

- 卡片管理
  - 新增、编辑、软删除、恢复、彻底删除
- 分类视图
  - 总览、到账单日、未到账单日、未结算卡、已还完卡
- 账单模式
  - 按账单月切换查看与操作
- 子页面操作
  - 还款、刷卡、流水记录、撤销流水
- 状态标记
  - 收费/未收费、VIP 客户、强制已还完
- 统计能力
  - 查看当日刷卡/还款统计、查看账单总额
- 导出能力
  - 生成账单 Excel 并本地落盘

## 4. 测试视角下的高风险点

- 发布包能否在新机器稳定启动
- 页面按钮、弹窗和输入框是否正常工作
- 账单月切换后是否出现串月
- 分类判定和收费状态联动是否正确
- 刷卡、还款、撤销后的金额是否完整联动
- 流水时间是否记录为本地实际时间
- 导出文件内容、路径和格式是否可靠
- 缓存和编码问题是否影响最终展示结果

## 5. 真实历史问题方向

项目在真实使用中曾出现过以下问题：

- 双击启动后页面不显示，或启动超时
- “查看刷卡/还款”按钮点击无反应
- 关闭账单弹窗后页面黑屏
- 还款/刷卡输入框不能输入数字
- 选择 3 月账单时显示了不属于 3 月周期的卡
- 2 月账单出现“未到账单日”异常
- 切换收费状态后卡片跳错分类
- 强制已还完入口缺失或点击无效
- 3 月 1 日操作却显示为 3 月 16 日
- 同一卡从不同分类进入子页面时记录不一致
- 撤销流水后金额回滚不完整
- 导出只有标题没有数据，或因为路径非法直接失败
- 浏览器缓存导致列错位、页面中文乱码
