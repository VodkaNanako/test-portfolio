﻿# 核心测试用例展开

## 1. 文档说明

本文档用于补充《测试用例清单.md》中的展示版总览，选取最能体现测试设计能力、业务规则理解能力和回归测试价值的核心用例进行展开。

本文件侧重展示：

- 如何从业务规则拆测试点
- 如何设置前置条件与测试数据
- 如何验证页面、接口、日志和数据库的一致性
- 如何围绕真实历史缺陷设计回归用例

---

## 2. 用例字段说明

- 用例编号：唯一标识
- 模块：所属功能或规则区域
- 优先级：`P0` / `P1` / `P2`
- 前置条件：执行前需满足的环境、数据或状态
- 测试步骤：执行路径
- 测试数据：关键输入值
- 预期结果：页面、接口、日志、数据库等维度的预期表现
- 关键校验点：本用例最核心的验证对象

## 3. 核心测试用例

### CASE-001 历史账单分类稳定

- 用例编号：`MONTH-EXP-001`
- 模块：账单月 / 分类逻辑
- 优先级：`P0`

#### 前置条件
1. 系统中至少存在一张 2 月账单卡片
2. 本地系统日期已进入 3 月
3. 2 月账单已有对应日志和状态数据

#### 测试步骤
1. 启动系统并进入账单月视图
2. 切换账单月为 `2026-02`
3. 查看 2 月账单卡片的分类归属
4. 切换总览与分类视图重复检查

#### 测试数据
- `billMonth = 2026-02`
- 指定存在历史账单数据的卡片

#### 预期结果
1. 2 月账单卡片仍按照 2 月账单规则分类
2. 不出现“未到账单日”等基于当前日期误判的分类结果
3. 总览与分类视图中的分类结果一致
4. 页面展示与详情数据一致

#### 关键校验点
- 历史账单是否被当前日期污染
- 历史账单分类规则是否稳定
- 跨视图结果是否一致

### CASE-002 切换收费状态不跳错分类

- 用例编号：`MONTH-EXP-002`
- 模块：收费状态联动 / 分类逻辑
- 优先级：`P0`

#### 前置条件
1. 存在至少一张可操作卡片
2. 当前账单月已生成月份配置数据
3. 卡片当前处于可观察分类的状态

#### 测试步骤
1. 进入指定账单月视图
2. 记录当前卡片所在分类
3. 切换该卡收费状态（收费/未收费）
4. 观察当前列表与分类视图变化
5. 再次进入详情页检查对应月份状态

#### 测试数据
- `billMonth = 2026-04`
- 指定可切换收费状态的卡片

#### 预期结果
1. 收费状态切换成功
2. 卡片不会因为切换状态而跳到错误分类
3. 当前账单月状态更新正确，其他月份不受影响
4. 页面分类、详情数据与月份状态表保持一致

#### 关键校验点
- 状态联动是否只影响标记本身
- 是否发生跨分类误跳
- 是否发生跨月份串数据

---

### CASE-003 刷卡金额联动

- 用例编号：`FLOW-EXP-001`
- 模块：刷卡 / 金额联动 / 日志
- 优先级：`P0`

#### 前置条件
1. 指定卡片存在且未删除
2. 指定账单月可正常操作
3. 当前卡片可用额度满足本次刷卡金额
4. 已记录刷卡前的详情数据和日志数量

#### 测试步骤
1. 调用刷卡操作，输入合法金额
2. 返回主界面或详情页查看金额变化
3. 打开流水记录查看新增日志
4. 通过接口或数据库回查该卡当前账单月数据

#### 测试数据
- `cardId = 1`
- `billMonth = 2026-04`
- `amount = 100`

#### 预期结果
1. 刷卡操作成功
2. `spentAmount` 按预期增加
3. 未还金额、未刷金额按业务规则联动变化
4. 日志新增一条刷卡记录，金额和账单月正确
5. 页面展示、接口返回、数据库日志一致

#### 关键校验点
- 金额联动是否正确
- 日志是否新增且字段正确
- 页面、接口、数据库三层结果是否一致

### CASE-004 撤销操作回滚

- 用例编号：`FLOW-EXP-002`
- 模块：撤销流水 / 金额回滚
- 优先级：`P0`

#### 前置条件
1. 指定卡片已有至少一条可撤销刷卡或还款日志
2. 已记录撤销前金额、日志和状态数据
3. 当前账单月下存在对应流水记录

#### 测试步骤
1. 选择一条已存在的刷卡或还款日志
2. 执行撤销操作
3. 检查日志列表是否删除该记录
4. 检查相关金额字段是否回滚
5. 通过接口或数据库再次确认结果

#### 测试数据
- `cardId = 1`
- `billMonth = 2026-04`
- `logId = 指定待撤销流水`

#### 预期结果
1. 撤销成功
2. 被撤销日志从列表中移除
3. 已刷金额 / 已还金额 / 未还金额等字段完整回滚
4. 页面展示、接口返回与数据库记录一致
5. 其他未撤销日志不受影响

#### 关键校验点
- 撤销是否只作用于目标流水
- 金额回滚是否完整
- 是否存在部分字段未回滚风险

### CASE-005 导出路径合法

- 用例编号：`EXPORT-EXP-001`
- 模块：导出功能 / 文件路径
- 优先级：`P0`

#### 前置条件
1. 指定账单月存在可导出数据
2. 系统可执行本地导出
3. 当前运行目录包含历史容易触发异常的目录名或路径条件

#### 测试步骤
1. 进入导出功能页面或执行本地导出接口
2. 选择指定账单月执行导出
3. 观察导出过程是否报错
4. 检查生成文件的名称与落盘路径
5. 打开导出文件确认内容可读

#### 测试数据
- `billMonth = 2026-04`
- 历史异常目录场景

#### 预期结果
1. 导出过程无 `Illegal char <>` 等路径错误
2. 文件成功生成
3. 文件名和路径合法、可预期
4. 文件内容可正常打开并包含数据

#### 关键校验点
- 导出路径是否经过合法化处理
- 文件是否真正生成成功
- 导出结果是否不仅“无报错”，且文件可用

## 4. 设计结论

本文件中的 5 条展开用例，分别对应本项目最能体现测试能力的 5 类场景：

- 历史账单规则稳定性

- 高风险撤销回滚
- 导出与交付质量验证
