# 回归测试清单

## 1. 文档说明

本文档用于版本发布前的最小回归验证，重点覆盖项目中真实出现过的高风险历史缺陷，确保以下问题不再复发：

- 发布包不可用
- 核心按钮和输入控件失效
- 账单月与分类规则错误
- 金额、时间和日志联动异常
- 导出、缓存和编码问题影响交付

## 2. 回归环境建议

- 操作系统：Windows
- 运行方式：
  - 发布包模式：双击 `启动网页版.bat`
  - 开发模式：`java -jar target/credit-card-manager-0.0.1-SNAPSHOT.jar`
- 浏览器：Chrome
- 数据库：`data/credit.db`

## 4. 推荐回归顺序

建议每次发版按以下顺序执行，优先保障可交付性：

1. 发布包启动与访问
2. 核心按钮、输入框和弹窗交互
3. 账单月与分类规则
4. 刷卡、还款、撤销、时间记录
5. 导出、路径、落盘位置
6. 缓存、编码和展示一致性

## 5. 发布前最小必跑集

如果时间有限，至少执行以下回归项：

- `REG-001` 发布包启动
- `REG-003` 核心按钮交互
- `REG-005` 子页面输入
- `REG-006` 账单月判定
- `REG-007` 历史账单分类
- `REG-008` 收费状态联动
- `REG-011` 时间记录
- `REG-013` 撤销回滚
- `REG-014` Excel 内容完整性
- `REG-016` Excel 路径合法性
- `REG-018` 缓存错位

## 6. 回归结论填写建议

每次执行回归后，建议按以下结果填写：

- `Pass`：结果符合预期
- `Fail`：结果不符合预期，需提缺陷
- `Block`：前置条件异常，当前项无法继续
- `Skip`：本轮不适用

如果发现失败项，建议同步补充：

- 缺陷编号
- 复现步骤
- 影响范围
- 是否阻断发布
